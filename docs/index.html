Persistent Memory
Claude remembers your training context
Without memory, Claude forgets everything between conversations. With strava-mcp-agent, your zones, pace trends, injuries, and goals persist across every session.
Without memory
Session 1: "Your Z2 pace is 9:30/km at 150bpm"
Session 2: "Your Z2 pace is 8:10/km at 150bpm"
Session 3: "I'd recommend running at 9:00-9:30/km for easy runs" ← wrong! forgot the improvement
With memory
Session 1: Z2 pace saved: 9:30/km
Session 2: Z2 pace auto-updated: 8:10/km
Session 3: "Your Z2 pace has improved to 8:10/km — run at 8:00-8:30 for easy runs" ✓ accurate

Your weekly mileage is trending up from 32 to 38 km. Given your sub-50 10K goal in June, you're right on track. I'd suggest bumping your long run to 14 km next week.

Quick Start
Running in 2 commands
Install, run the setup wizard, and you're done. It auto-configures Claude Desktop for you.

1. Install from PyPI

One command. No extra dependencies to manage.

2. Create a Strava API app (one time)

Go to strava.com/settings/api and fill in:

Application NameAnything (e.g. My Claude MCP)
CategoryPick any
ClubLeave blank
Websitehttp://localhost
Authorization Callback Domainlocalhost ← important!

Click Create. Copy your Client ID (a number) and Client Secret (a long code) from the next page.

3. Run the setup wizard

Run strava-mcp-token in your terminal. It will ask for your Client ID and Secret, open your browser to authorize with Strava, then automatically configure Claude Desktop — detects your OS, finds the right Python, and writes the config file for you.

4. Restart Claude Desktop

That's it. Your 13 Strava tools are ready to use.

Terminal
# Install
$ pip install strava-mcp-agent

# Setup wizard — handles everything
